# 更新日志

## v2.5.0 (2026-05-27)

### 新增
- 新增 AstrBot WebUI 插件页面「图片历史」，可查看已成功发送的 Pixiv 图片记录
- 图片历史支持本地缩略图、网格/列表视图、关键词、来源、R18 和会话类型筛选
- 发送记录覆盖搜索、排行榜、作品下载和今日运势配图，只有发送成功后才写入
- 新增 SQLite 当天图片索引，用于普通发图和今日运势配图避免重复

### 改进
- 新增 `Pillow` 依赖用于生成本地缩略图
- 原始发送图片仍按原逻辑清理，仅保留缩略图和元数据
- 图片历史默认保留最近 200 条，淘汰记录时同步清理旧缩略图
- 普通发图去重改为按北京时间当天持久保存，插件重启后仍有效，次日自动清理
- 今日运势同一用户同一天在同一群聊/私聊中的文本和配图都会持久固定，模型文案生成后不再因配置变化而改变
- 今日运势模型文案提示词改为内置固定模板，不再暴露配置项

### 修复
- 修复今日运势配图并发触发时跨用户可能选中同一作品的问题，改为发送前原子占用，发送失败后自动释放
- 修复已固定的今日运势配图重新发送时丢失原始来源，导致后续成功发送无法写入当天共用去重索引的问题
- 修复 SQLite 连接未显式关闭时，Windows 下测试或清理临时数据库可能遇到文件占用的问题

### 测试
- 补充 `ImageIndexStore` 的原子占用、释放、今日运势同日固定、来源持久化、次日清理和并发 claim 回归测试

## v2.4.0 (2026-05-25)

### 新增
- 新增今日运势功能，支持 `/今日运势`、`/jrys` 和纯文本 `今日运势`、`jrys` 触发
- 今日运势支持调用文本模型根据抽取结果生成文案，模型失败时自动回退预置文案
- 今日运势支持 Pixiv 配图，配图获取或发送失败时自动回退纯文字
- 同一用户同一天在同一群聊/私聊中的运势结果保持稳定
- 今日运势配图在同一天同群聊内会优先避开已发送作品，尽量让不同用户拿到不同配图
- 新增 `/p` 近期去重，同一群聊同一标签/排行榜会优先发送近期未出现过的作品

### 配置
- 新增 `fortune_enabled` 配置项（今日运势开关）
- 新增 `fortune_image_enabled` 配置项（今日运势 Pixiv 配图开关）
- 新增 `fortune_image_tag` 配置项（今日运势配图标签，支持英文逗号分隔多个标签，留空使用默认排行榜）
- 新增 `fortune_ai_text_enabled`、`fortune_ai_provider_id`、`fortune_ai_prompt` 配置项（今日运势模型文案）
- 新增 `fortune_ai_timeout_seconds` 配置项（今日运势模型文案超时，超时回退预置文案）
- 优化 `fortune_ai_prompt` 默认提示词，改为固定格式输出运势文案
- 新增 `dedupe_ttl_hours` 配置项（近期去重保留时间）

## v2.3.0 (2026-05-25)

### ✨ 新增
- **自然语言自动触发**（需在配置中开启 `auto_trigger_enabled`）
  - 群内发送「来一份图」「来三张初音ミク图」等自然语言时自动触发 Pixiv 发图
  - 支持中文数字（一到十、两）和阿拉伯数字
  - 支持标签搜索和排行榜（无标签时）
  - 受频率限制保护

### 🔧 改进
- **模块拆分**：将 PixivClient、ImageDownloader、AiCommenter 拆分为独立模块
  - `pixiv_client.py` — Pixiv API 客户端
  - `downloader.py` — 图片下载、URL 提取、临时文件管理
  - `ai_commenter.py` — AI 识图评论管道
- 减少代码重复，提升可维护性

### 🐛 修复
- 修复自动触发未生效：`super().__init__` 需传入 `config` 参数
- 移除自动触发中的 `event.stop_event()` 调用
- 正则改为必须以「图」结尾，避免「来份外卖」等误触发

### 📝 配置
- 新增 `auto_trigger_enabled` 配置项（自然语言自动触发开关）

## v2.2.0 (2026-05-24)

### ✨ 新增
- **发送重试机制**：图片发送失败时自动重试，最多3次
  - 递增等待时间（2秒、4秒）
  - 适用于所有发送场景：搜索结果、排行榜、作品下载
  - 详细的重试日志记录
- **合并转发失败降级**：合并转发失败后自动降级为逐条发送
- **失败提示内嵌合并消息**：下载失败的作品ID显示在合并消息末尾
- **日志友善提示**：发送失败时日志显示友善提示+原始错误，用户端保持原样

## v2.1.0 (2026-05-24)

### ✨ 新增
- **作品ID下载**：`/pd <作品ID> [页码]` 通过作品ID直接下载并发送图片
  - 支持多页作品指定页码：`/pd 12345678 2`（下载第2页）
  - 自动 R18 内容过滤检查
  - 原图过大时自动降级到低质量图片
  - 支持 AI 识图评论（如果启用）

### 🔧 改进
- 帮助文档新增 `/pd` 命令说明
- 版本号更新至 v2.1.0

## v2.0.0 (2026-05-23)

### ✨ 新增
- **PixivClient 类**：封装 API 认证、缓存、自动刷新逻辑
- **排行榜支持**：8 种排行类型，`/pv rank list` 查看详情
- **作品详情**：`/pv info <ID>` 查看标题/作者/标签/浏览量/收藏数
- **帮助指令**：`/pv help` 显示使用说明
- **指令别名**：支持 `/pixiv`、`/搜图`
- **请求频率限制**：防刷屏，可配置间隔秒数
- **多页作品支持**：正确获取多页作品图片
- **平台限定**：仅支持 aiocqhttp（OneBot）平台

### 🔧 改进
- **重构为 PixivClient 类**：API 逻辑与插件逻辑分离
- **全局 aiohttp 会话**：异步安全，正确的生命周期管理
- **完善的错误处理**：分类错误提示（超时/HTTP错误/网络异常）
- **详细的日志输出**：统一 `[GetPx]` 前缀
- **完善的 terminate 清理**：正确关闭 API 和 HTTP 会话
- **发送前提示**：每张图片附带标题和作品ID
- **频率限制反馈**：告知用户需等待秒数

### 📝 配置
- 新增 `rate_limit_seconds` 配置项（请求频率限制）
- 更新 `pixiv_ranking_mode` 提示信息
