### 修复
- 修复 `dedupe_ttl_hours` 数值仅被当作开关、名称与实际行为不一致的问题；新增最近 `0–7` 个北京时间自然日去重，并在缩短窗口时清理超期索引。
- 修复 AI 问候错误角色或单标记 Markdown 可能被保存、远程问候升级写入失败会中断签到卡、管理员预览问候异常会直接中断的问题。
- 修复去重清理失败后仍被限频、配置窗口未回滚，以及图片下载部分临时文件和文件描述符异常路径未释放的问题。
- 修复关闭去重时签到发送失败会错误撤销背景、昵称缺失时用户 ID 进入 AI 问候上下文，以及无缓存背景刷新和画质探测残留 T2I 临时文件的问题。
- 修复修改签到卡画质后主动刷新背景仍复用当天旧档位的问题；恢复普通作品最终下载和逐条发送成功的 `info` 日志。
- 修复关闭去重时签到消息发送失败会错误触发背景占用撤销的问题。

### 优化
- 优化 AI 问候数据传递：移除值为"无"的空字段，精简提示词并明确说明只使用存在的信息。
- 提取 `_prepare_configurable_instruction` 函数专门处理 AI 问候提示词模板清理，提升可读性。
- 提取 `_plan_download_attempts` 函数处理图片下载路由预算分配，降低嵌套深度。

### 新增
- 新增 `省流量`、`清晰`、`极致` 三档签到卡画质，渲染失败时按档位降级并使用精确尺寸缓存校验。
- 新增多个 Lolicon 图片反代 origin 的顺序轮换、原地址兜底和质量降级；不影响 Pixiv 登录直连策略。

### 变更
- 移除独立 `checkin_card_quality` JPEG 质量配置，签到卡统一使用 JPEG 质量 95。
- 旧 `dedupe_ttl_hours` 从配置界面隐藏，并一次性迁移为新的 `dedupe_days`。
- 去重、签到渲染缓存、图片反代、AI 问候和一言统一增加脱敏结构化结果日志；普通作品最终下载和发送成功使用 `info`，其他高频成功与格式拒绝使用 `debug`，故障告警按原因限频，不记录提示词、回复正文、完整 URL 或用户标识。
- 签到命令改为 `签到我的`、`签到排行`、`签到商店`、`签到管理` 四个独立指令组，移除旧 `签到中心` 路由，并重新生成 `/签到帮助` 帮助图。

### 升级说明
- **AI 问候优化**：系统现在只会传入有效信息，不再传入"无"的空字段。如果你使用了自定义 AI 问候提示词（`checkin_ai_greeting_prompt`），建议在提示词末尾添加："根据提供的数据生成问候语，只使用存在的信息，缺失的信息不必提及。"

- **排行榜支持**：8 种排行类型，`/prl` 查看详情
- **作品详情**：`/pi <ID>` 查看标题/作者/标签/浏览量/收藏数
- **帮助指令**：`/ph` 显示使用说明
- **短命令入口**：当前使用 `/p`、`/pr`、`/prl`、`/pi`、`/pd`、`/ph`
- **请求频率限制**：防刷屏，可配置间隔秒数
- **多页作品支持**：正确获取多页作品图片
- **平台限定**：仅支持 aiocqhttp（OneBot）平台

### 🔧 改进
- **重构为 PixivClient 类**：API 逻辑与插件逻辑分离
- **全局 aiohttp 会话**：异步安全，正确的生命周期管理
- **完善的错误处理**：分类错误提示（超时/HTTP错误/网络异常）
- **详细的日志输出**：统一 `[GetPx]` 前缀
- **完善的 terminate 清理**：正确关闭 API 和 HTTP 会话
- **发送前提示**：每张图片附带标题和作品ID
- **频率限制反馈**：告知用户需等待秒数

### 📝 配置
- 新增 `rate_limit_seconds` 配置项（请求频率限制）
- 更新 `pixiv_ranking_mode` 提示信息
