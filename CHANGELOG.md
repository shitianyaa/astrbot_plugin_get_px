# 更新日志

## v2.6.0 (2026-06-03)

### 新增
- 新增签到自定义背景完整性校验，损坏或非图片文件会被跳过。
- 新增下载大小上限保护，超出 50 MiB 的图片会被拒绝。

### 改进
- 图片下载改为流式写入临时文件，避免大图响应整体进入内存。
- 原图超过自动降级阈值时继续清理旧临时文件，降低多图发送时的资源压力。
- 图片历史 JSON 损坏时会备份为 `.corrupt.<timestamp>` 文件，再以新历史继续写入，避免静默覆盖损坏原件。
- Pixiv 作品详情只在明确缺失时返回“未找到”，网络、登录或 API 异常会向命令层传递并记录日志。
- WebUI 图片历史和黑名单 API 的内部错误响应改为通用文案，详细异常仅写入日志。
- 已核验 AstrBot 插件页面文档：Bridge 会转发到 `/api/plug/<plugin_name>/...`，文档只要求插件名前缀，未发现插件侧额外鉴权 API；本版不新增自定义鉴权层。

### 修复
- 修正配置提示中的旧排行榜命令为 `/prl`。
- 修正 `/ph` 帮助中的自然语言示例，补上结尾“图”。
- `metadata.yaml` 的依赖声明与 `requirements.txt` 对齐。

## v2.5.0 (2026-05-27)

### 新增
- 新增 AstrBot WebUI 插件页面「图片历史」，可查看已成功发送的 Pixiv 图片记录
- 图片历史支持本地缩略图、网格/列表视图、关键词、来源、R18、会话类型筛选和作品黑名单
- 发送记录覆盖搜索、排行榜、作品下载和签到 Pixiv 背景，只有发送成功后才写入
- 新增 SQLite 当天图片索引，用于普通发图避免重复
- 新增每日签到，支持 HTML 卡片、Pixiv/自定义背景、金币、好感度、连续签到和好感度双倍加持
- 新增管理员命令 `/签到测试`，可预览签到卡片和背景且不写入签到数据

### 改进
- 新增 `Pillow` 依赖用于生成本地缩略图
- 原始发送图片仍按原逻辑清理，仅保留缩略图和元数据
- 图片历史默认保留最近 200 条，淘汰记录时同步清理旧缩略图
- 普通发图去重改为按北京时间当天持久保存，插件重启后仍有效，次日自动清理
- 签到背景沿用 R18 模式、拉黑标签和作品 ID 黑名单过滤
- 签到 Pixiv 背景会写入图片历史并保留本地缩略图

### 修复
- 修复 SQLite 连接未显式关闭时，Windows 下测试或清理临时数据库可能遇到文件占用的问题

### 测试
- 补充 `ImageIndexStore` 的原子占用、释放、次日清理和黑名单回归测试

## v2.3.0 (2026-05-25)

### ✨ 新增
- **自然语言自动触发**（需在配置中开启 `auto_trigger_enabled`）
  - 群内发送「来一份图」「来三张初音ミク图」等自然语言时自动触发 Pixiv 发图
  - 支持中文数字（一到十、两）和阿拉伯数字
  - 支持标签搜索和排行榜（无标签时）
  - 受频率限制保护

### 🔧 改进
- **模块拆分**：将 PixivClient、ImageDownloader、AiCommenter 拆分为独立模块
  - `pixiv_client.py` — Pixiv API 客户端
  - `downloader.py` — 图片下载、URL 提取、临时文件管理
  - `ai_commenter.py` — AI 识图评论管道
- 减少代码重复，提升可维护性

### 🐛 修复
- 修复自动触发未生效：`super().__init__` 需传入 `config` 参数
- 移除自动触发中的 `event.stop_event()` 调用
- 正则改为必须以「图」结尾，避免「来份外卖」等误触发

### 📝 配置
- 新增 `auto_trigger_enabled` 配置项（自然语言自动触发开关）

## v2.2.0 (2026-05-24)

### ✨ 新增
- **发送重试机制**：图片发送失败时自动重试，最多3次
  - 递增等待时间（2秒、4秒）
  - 适用于所有发送场景：搜索结果、排行榜、作品下载
  - 详细的重试日志记录
- **合并转发失败降级**：合并转发失败后自动降级为逐条发送
- **失败提示内嵌合并消息**：下载失败的作品ID显示在合并消息末尾
- **日志友善提示**：发送失败时日志显示友善提示+原始错误，用户端保持原样

## v2.1.0 (2026-05-24)

### ✨ 新增
- **作品ID下载**：`/pd <作品ID> [页码]` 通过作品ID直接下载并发送图片
  - 支持多页作品指定页码：`/pd 12345678 2`（下载第2页）
  - 自动 R18 内容过滤检查
  - 原图过大时自动降级到低质量图片
  - 支持 AI 识图评论（如果启用）

### 🔧 改进
- 帮助文档新增 `/pd` 命令说明
- 版本号更新至 v2.1.0

## v2.0.0 (2026-05-23)

### ✨ 新增
- **PixivClient 类**：封装 API 认证、缓存、自动刷新逻辑
- **排行榜支持**：8 种排行类型，`/prl` 查看详情
- **作品详情**：`/pi <ID>` 查看标题/作者/标签/浏览量/收藏数
- **帮助指令**：`/ph` 显示使用说明
- **短命令入口**：当前使用 `/p`、`/pr`、`/prl`、`/pi`、`/pd`、`/ph`
- **请求频率限制**：防刷屏，可配置间隔秒数
- **多页作品支持**：正确获取多页作品图片
- **平台限定**：仅支持 aiocqhttp（OneBot）平台

### 🔧 改进
- **重构为 PixivClient 类**：API 逻辑与插件逻辑分离
- **全局 aiohttp 会话**：异步安全，正确的生命周期管理
- **完善的错误处理**：分类错误提示（超时/HTTP错误/网络异常）
- **详细的日志输出**：统一 `[GetPx]` 前缀
- **完善的 terminate 清理**：正确关闭 API 和 HTTP 会话
- **发送前提示**：每张图片附带标题和作品ID
- **频率限制反馈**：告知用户需等待秒数

### 📝 配置
- 新增 `rate_limit_seconds` 配置项（请求频率限制）
- 更新 `pixiv_ranking_mode` 提示信息
